# Put device in maskrom/loader mode (usually by holding recovery button while powering on)

# Download the boot loader
sudo rkdeveloptool db /path/to/rkbin/rk356x_spl_loader_v1.xx.bin

# Write boot partition
sudo rkdeveloptool wl 0x4000 Volume/build-rockchip-rk3562-evb/tmp/deploy/images/rockchip-rk3562-evb/boot.img

# Write rootfs (you'll need to extract it first)
cd Volume/build-rockchip-rk3562-evb/tmp/deploy/images/rockchip-rk3562-evb/
mkdir -p /tmp/rootfs
sudo tar -xzf core-image-minimal-rockchip-rk3562-evb.rootfs.tar.gz -C /tmp/rootfs

# Create ext4 image from rootfs
dd if=/dev/zero of=/tmp/rootfs.img bs=1M count=512
mkfs.ext4 /tmp/rootfs.img
sudo mount /tmp/rootfs.img /mnt
sudo cp -a /tmp/rootfs/* /mnt/
sudo umount /mnt

# Flash rootfs
sudo rkdeveloptool wl 0x8000 /tmp/rootfs.img

# Reboot device
sudo rkdeveloptool rd

# Loads loader to RAM
$ sudo upgrade_tool db rk3562_spl_loader_v1.07.106.bin

# Boot from RAM loader!
$ sudo upgrade_tool ul rk3562_spl_loader_v1.07.106.bin

# Writes U-Boot to eMMC
$ sudo upgrade_tool wl 0x40 rk3562_spl_loader_v1.07.106.bin
$ sudo upgrade_tool wl 0x4000 uboot.img


### What Each Command Does

1. **`db` (download boot)**: Downloads the loader to device RAM (doesn't write to flash)
   - Device enters maskrom/loader mode
   - Loader runs from RAM temporarily

2. **`wl` (write LBA)**: Writes U-Boot image to eMMC at offset 0x4000
   - This is U-Boot proper, not the early boot stages
   - Offset 0x4000 = sector 0x4000 (8KB offset)

3. **`ul` (upgrade loader)**: Actually writes the full loader chain to eMMC **AND** boots from it
   - Writes TPL/SPL/loader to proper offsets (0x40, 0x4000, etc.)
   - Includes the idbloader.img equivalent
   - Kicks off the boot sequence

==================================

ping command
    |
    └─> net_ping()
            |
            └─> net_send_packet()
                    |
                    └─> eth->send()  [eth_ops]
                            |
                            └─> eqos_send()  [MAC driver]
                                    |
                                    └─> DMA descriptor setup
                                            |
                                            └─> Hardware transmission


Function Call Chain (Network RX):
==================================

eth->recv()  [polling or IRQ]
    |
    └─> eqos_recv()  [MAC driver]
            |
            └─> DMA descriptor check
                    |
                    └─> net_receive()  [NET core]
                            |
                            └─> Protocol handler (IP/ARP/etc)
